EXERCISE 2: E-COMMERCE PLATFORM SEARCH FUNCTION

1. Understanding Asymptotic Notation

Big O notation is used to measure the efficiency of an algorithm by describing how its execution time grows as the input size increases. It helps compare different algorithms and choose the most efficient one for large datasets.

Common Time Complexities:
• O(1) – Constant Time
• O(log n) – Logarithmic Time
• O(n) – Linear Time
• O(n log n) – Linearithmic Time
• O(n²) – Quadratic Time

Search Operation Scenarios:

Linear Search:
• Best Case: O(1) – Element found at the first position.
• Average Case: O(n) – Element found in the middle.
• Worst Case: O(n) – Element found at the last position or not found.

Binary Search:
• Best Case: O(1) – Element found at the middle position.
• Average Case: O(log n)
• Worst Case: O(log n)

2. Implementation

Product Class:
Stores product information such as:
• productId
• productName
• category

Linear Search:
Linear search checks each element one by one until the required product is found.

Binary Search:
Binary search repeatedly divides a sorted array into two halves and searches only the relevant half.

Requirements:
• Linear Search uses an unsorted array.
• Binary Search requires a sorted array.

3. Time Complexity Analysis

Linear Search:

Best Case: O(1)
Average Case: O(n)
Worst Case: O(n)

Binary Search:

Best Case: O(1)
Average Case: O(log n)
Worst Case: O(log n)

4. Comparison

Linear Search:
Advantages:
• Simple to implement.
• Works on unsorted data.

Disadvantages:
• Slow for large datasets.

Binary Search:
Advantages:
• Much faster for large datasets.
• Efficient searching.

Disadvantages:
• Requires sorted data.

5. Suitable Algorithm for E-commerce Platform

Binary Search is more suitable for an e-commerce platform because product catalogs can contain thousands or millions of products. Its O(log n) time complexity provides significantly faster search performance compared to the O(n) complexity of linear search.

Conclusion:

Linear Search is useful for small or unsorted datasets. Binary Search is preferred for large, sorted datasets because it offers much better performance and scalability.
